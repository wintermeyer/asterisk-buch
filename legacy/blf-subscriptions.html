23.2. Verfahren für SIP-Subscriptions

Anmerkung

Dieser Abschnitt ist ein kleiner Exkurs für die Technikinteressierten. Für die Konfiguration ist er in der Regel nicht weiter von Bedeutung. Sie können also getrost zum nächsten Abschnitt springen.
Es gibt in SIP mehrere unterschiedlich Verfahren, wie man den Status einer Nebenstelle „abonnieren“ kann. Die gebräuchlichste SIP-Erweiterung dafür ist „Dialog Event Package“ (RFC 4235), an die sich u. a. auch die 3x0-Snoms halten. Dieser Standard basiert auf „SIP Event Notification“ (RFC 3265). Dabei schickt das Telefon für jede zu überwachende Nebenstelle ein SUBSCRIBE an die Telefonanlage und erhält bei jeder Status-Änderung ein NOTIFY (PUBLISH wird von Asterisk noch nicht unterstützt). Der Hauptunterschied sind die unterschiedlichen (XML-)Formate im „Body“ der „Notifications“.
Die von Asterisk (mehr oder weniger) unterstützten Verfahren sind:
SpezifikationMIME-TypGeräte
SIP Dialog Event Package (RFC 4235)application/dialog-info+xmlSnom, Linksys, Aastra, Polycom, Grandstream, Thomson, ...
Presence Information Data Format (PIDF) (RFC 3863)application/pidf+xmlCounterPath[a] EyeBeam etc., Cisco
Vor-Version von RFC 3863 (PIDF, s. o.), XPIDF, CPIM mit Microsoft-Ergänzung[b]application/xpidf+xmlMicrosoft RTC/LCS, MSN Messenger, Polycom
Vor-Version von RFC 3863 (PIDF, s. o.), XPIDF, CPIMapplication/cpim-pidf+xml?

[a] früher Xten

[b] XPIDF Presence Format Example, XPIDF Presence Document Format, Microsoft Real Time Communication (RTC) Client
