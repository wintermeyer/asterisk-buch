[[feedback]]
= Mitwirken und Feedback
:description: Mitmachen am Asterisk-Buch: Pull Requests, Issues, E-Mail-Feedback.

Dieses Buch ist kein abgeschlossener Text, es soll "lebendig" bleiben und
ständig verbessert werden. Der gesamte Text steht unter der GNU Free
Documentation License (siehe xref:gnu-fdl.adoc[_GNU Free Documentation
License_]). Die jeweils aktuelle Version finden Sie unter
https://das-asterisk-buch.de.

Wenn Sie Verbesserungsvorschläge haben, gibt es zwei bequeme Wege:

* *Pull Request* im Git-Repository. Die Seiten liegen als AsciiDoc (Antora)
  unter `modules/ROOT/pages/`. Ein PR ist die schnellste Variante, wenn
  Sie den Fehler selbst sehen und beheben können.
* *E-Mail* an mailto:sw@wintermeyer-consulting.de[sw@wintermeyer-consulting.de]
  oder ein Issue im Repository. Bitte mit Angabe der Seite/URL und, falls
  zutreffend, Ihrer Asterisk-Version.

Schon an dieser Stelle: vielen Dank für Ihr Interesse und Ihr Engagement.

[[Arten-der-Hilfe]]
== Arten der Hilfe

Einem Open-Source-Projekt wie diesem können Sie auf verschiedene Arten
helfen: eine Review schreiben, einen Typo-Fix als PR einreichen, ein
ganzes Kapitel beisteuern, oder einfach im Kollegenkreis darauf
hinweisen, dass es dieses Buch gibt.

[[beta-test-faq]]
== FAQ für freiwillige Helfer

[cols="2*"]
|===
| 1. Was sollte gemeldet werden? |
| *F:* | Ich habe etwas aus dem Buch ausprobiert, und es funktioniert nicht so wie beschrieben. Melden?
| *A:* | Ja, unbedingt — am besten als Issue oder PR mit Angabe der Seite/URL, Ihrer Asterisk-Version und der Fehlermeldung.
| *F:* | Wie melde ich einen Fehler?
| *A:* | GitHub-Issue am Repository oder E-Mail an mailto:sw@wintermeyer-consulting.de[sw@wintermeyer-consulting.de].
| 2. Fragen zur Technik |
| *F:* | In welchem Format wird das Buch geschrieben?
| *A:* | Das Buch wird als https://asciidoctor.org/[AsciiDoc] geschrieben und mit https://antora.org/[Antora] gebaut. Die Seiten liegen unter `modules/ROOT/pages/`.
| *F:* | Mit welchem Editor kann ich AsciiDoc schreiben?
| *A:* | Jeder Texteditor reicht — VS Code mit der AsciiDoc-Extension, Neovim, Emacs. Für Live-Preview bieten sich https://asciidoctor.org/docs/editors/[die gängigen IDE-Plugins] an.
| *F:* | Wie wird die Webseite generiert?
| *A:* | Bei jedem Push auf `main` läuft ein GitHub-Actions-Workflow, der mit Antora das Buch baut und unter https://das-asterisk-buch.de veröffentlicht.
|===
