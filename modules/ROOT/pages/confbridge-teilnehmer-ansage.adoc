[[confbridge-teilnehmer-ansage]]
= Anzahl der Teilnehmer ansagen
:description: Anzahl der Teilnehmer einer ConfBridge-Konferenz mit CONFBRIDGE_INFO() ansagen.

Unter ConfBridge ist die Teilnehmerzahl nicht über eine
`ConfBridge()`-Option abrufbar, sondern über die Funktion
`CONFBRIDGE_INFO()`:

[source]
----
[intern]
exten => 5555,1,Answer()
 same => n,Wait(1)
 same => n,Set(TEILNEHMER=${CONFBRIDGE_INFO(parties,team)})
 same => n,Playback(conf-thereare)
 same => n,SayNumber(${TEILNEHMER})
 same => n,Playback(confbridge-participants)
 same => n,Hangup()
----

Die verketteten `conf-thereare` ("There are") und
`confbridge-participants` ("participants") sind in den mitgelieferten
_core-sounds-en_ enthalten. Für eine deutsche Ansage sollten Sie eigene
Prompts (aus xref:tts-text-to-speech.adoc[TTS] oder fertig
eingesprochen) hinterlegen.

`CONFBRIDGE_INFO(type,conference)` liefert je nach `type` verschiedene
Zahlen:

`parties`:: Gesamtzahl der Teilnehmer
`admins`:: Anzahl der Admins
`marked`:: Anzahl der "marked" Teilnehmer
`locked`:: `1`, wenn die Konferenz gesperrt ist

[TIP]
====
Das `Background()`-Menü "Zahl der aktuellen Teilnehmer durchgeben"
war früher mit MeetMes `c`-Option ein Einzeiler. Mit ConfBridge
müssen Sie die Zahl selbst aus `CONFBRIDGE_INFO()` lesen und per
`SayNumber()` ansagen — dafür haben Sie vollständige Kontrolle über
die Ansage (Sprache, Reihenfolge, zusätzliche Informationen).
====
