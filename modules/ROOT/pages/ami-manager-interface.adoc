[[ami-manager-interface]]
= Asterisk Manager Interface (AMI)
:description: Das Asterisk Manager Interface (AMI) als TCP-Protokoll für Monitoring und Steuerung.

In diesem Kapitel:

* xref:ami-konfiguration.adoc[Das Manager-Interface (AMI)]
* xref:ajam-deprecated.adoc[AJAM — und warum es durch ARI ersetzt wurde]

Das *Asterisk Manager Interface* (AMI) ist ein klassisches TCP-Protokoll
auf Port 5038. Ein externes Programm verbindet sich, meldet sich mit
Benutzername und Passwort an und kann ab dann:

* *Actions* senden — z. B. `Originate`, `Hangup`, `Redirect`,
  `MailboxCount`, `Command` (CLI-Befehl ausführen). Asterisk antwortet
  synchron mit einer `Response`.
* *Events* empfangen — ungefragt, asynchron: jeder neue Channel,
  jeder Bridge-Vorgang, jeder Hangup, jede Registrierung löst ein
  Event aus.

Das Protokoll ist zeilenbasiert mit `Key: Value`-Paaren, getrennt
durch CR/LF, ein Paket endet mit einer Leerzeile. Für
Monitoring-Skripte und Integrations-Werkzeuge reicht das völlig.

[NOTE]
====
Für neue Anwendungen, die Gespräche aktiv steuern sollen, empfehlen
wir heute xref:ari.adoc[ARI]: REST-API plus WebSocket-Event-Stream,
JSON statt zeilenbasiertem Text, sauberes Call-Control-Modell. AMI
bleibt relevant für Monitoring, Alt-Anwendungen und kleine Werkzeuge,
die einfach ein Event abgreifen wollen.
====

Die jeweils aktuelle Liste aller AMI-Actions und -Events steht in der
offiziellen Dokumentation unter
https://docs.asterisk.org/Asterisk_Latest_Documentation/API_Documentation/AMI_Actions/
und im CLI über `manager show commands` bzw. `manager show events`.
