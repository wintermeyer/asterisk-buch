[[blf-pjsip.conf]]
== pjsip.conf

Für BLF-Funktionalität müssen in `pjsip.conf` ein paar Optionen am
Endpoint gesetzt sein:

[source]
----
[21]
type=endpoint
context=interne-benutzer
disallow=all
allow=ulaw
allow=alaw
auth=21
aors=21
mailboxes=21@default          ; MWI
allow_subscribe=yes           ; Telefon darf SUBSCRIBE senden
sub_min_expiry=30
notify_early_inuse_ringing=yes
device_state_busy_at=0        ; Anzahl gleichzeitiger Gespräche, die als "busy" zählen
call_group=2                  ; Für Pickup
pickup_group=2
----

`device_state_busy_at=0` bedeutet: "Ab 0 parallelen Gesprächen gilt
der Endpoint als besetzt" — was effektiv "jedes Gespräch macht das
Telefon busy" heißt. Wer Transfers oder Zweitgespräche erlauben
will, setzt einen höheren Wert.

`call_group` und `pickup_group` regeln die Berechtigung beim
Heranholen: Ein Benutzer mit `pickup_group=2` darf Gespräche von
anderen Benutzern mit `call_group=2` übernehmen. Es gibt 64 Gruppen
(0–63).

[[blf-hints]]
== Hints

Als Nächstes muss man Asterisk die Zuordnung zwischen Extensions
und Benutzern (oder Geräten) mitteilen. Dies macht man im Dialplan mit
sogenannten „_Hints_“. Wenn man in
Dialplan-Prioritäten denkt, dann ist ein `hint` eine
spezielle Priorität, die noch vor der Priorität `1` steht.
Wichtig ist, dass sich die Hints in dem Context befinden, in dem der
überwachende Endpoint seine SUBSCRIBE-Anfragen auflösen darf
(also in dem Context, den Sie dem Endpoint in der `pjsip.conf`
zugewiesen haben). Andernfalls werden die Status-Informationen nicht
weitergeleitet. Das ist durchaus Absicht, damit man nicht per se von
allen Telefonen aus alle weiteren überwachen kann. Durch
unterschiedliche Contexte lässt sich also auch eine Art
Berechtigungssystem bauen.footnote:[Diese Methode eignet sich allerdings nicht so gut für datenbankgestützte „Realtime“-Installationen.] Wir beschreiben hier anhand von Beispielen, wie ein Hint
definiert wird. Je nachdem, ob Ihr Dialplan im
`extensions.conf`-Format oder in AEL geschrieben ist,
verwenden Sie die eine oder die andere Variante.

[cols="2*"]
|===
| `extensions.conf` | `extensions.ael`

| [interne-benutzer] exten => 21,*hint,PJSIP/21* exten => 21,1,Dial(PJSIP/$\{EXTEN},40) exten => 21,n,VoiceMail($\{EXTEN},u) exten => 22,*hint,PJSIP/22* exten => 22,1,Dial(PJSIP/$\{EXTEN},40) exten => 22,n,VoiceMail($\{EXTEN},u) | context interne-benutzer { *hint(PJSIP/21)* 21 => { Dial(PJSIP/$\{EXTEN},40); VoiceMail($\{EXTEN},u); } *hint(PJSIP/22)* 22 => { Dial(PJSIP/$\{EXTEN},40); VoiceMail($\{EXTEN},u); } }
|===

+

[cols="2*"]
|===
| `extensions.conf` | `extensions.ael`

| [interne-benutzer] exten => 21,*hint,PJSIP/21* exten => 22,*hint,PJSIP/22* exten => _2X,1,Dial(PJSIP/$\{EXTEN},40) exten => _2X,n,VoiceMail($\{EXTEN},u) | context interne-benutzer { *hint(PJSIP/21)* 21 => {} *hint(PJSIP/22)* 22 => {} _2X => { Dial(PJSIP/$\{EXTEN},40); VoiceMail($\{EXTEN},u); } }
|===

+

[IMPORTANT]
====
Beachten Sie hierbei, dass ein `hint` in Asterisk 1.4
nicht mit Patternfootnote:[xref:pattern-matching.adoc[Abschnitt 3.3, „Pattern Matching“\]] geschrieben werden kann.

[source]
----
exten => _2X,hint,PJSIP/${EXTEN}
----

bzw.

[source]
----
hint(PJSIP/${EXTEN}) _2X => {...}
----

wäre
also unzulässig.
Ab Asterisk 1.6 ist diese Schreibweise möglich, auch wenn es
anfänglich noch zu Problemen kam. Bitte beachten Sie aber, dass
während der Interpretation eines Hints noch kein Channel besteht, was
bedeutet, dass auch keine Channel-Variablen in der Hint-Definition zur
Verfügung stehen – mit Ausnahme von
`$\{EXTEN}`.
====

[cols="2*"]
|===
| `extensions.conf` | `extensions.ael`

| [interne-benutzer] exten => _2X,*hint,PJSIP/$\{EXTEN}* exten => _2X,1,Dial(PJSIP/$\{EXTEN},40) exten => _2X,n,VoiceMail($\{EXTEN},u) | context interne-benutzer { *hint(PJSIP/$\{EXTEN})* _2X => { Dial(PJSIP/$\{EXTEN},40); VoiceMail($\{EXTEN},u); } }
|===

+

[cols="2*"]
|===
| `extensions.conf` | `extensions.ael`

| [interne-benutzer] exten => 20,*hint,PJSIP/21&PJSIP/22* exten => 20,1,Dial(PJSIP/21&PJSIP/22,40) | context interne-benutzer { *hint(PJSIP/21&PJSIP/22)* 20 => {} 20 => { Dial(PJSIP/21&PJSIP/22,40); } }
|===

+

[TIP]
====
Der Übersichtlichkeit halber kann man alle Hints auch in einen
eigenen Context legen und diesen überall dort mit `include =>
meine-hints` einbinden, wo er benötigt wird. Dies ist vor allem
in Asterisk-Versionen vor 1.6 interessant.
====

Danach müssen Sie Asterisk neu starten (*+`/etc/init.d/asterisk
reload`+* oder *+`asterisk -rx 'module
reload'`+*).
